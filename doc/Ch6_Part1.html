6.1 — متطلبات تشغيل النظام

يتطلب تشغيل نظام Market Insight توافر مجموعة من المتطلبات التقنية على جانبَي الخادم والعميل لضمان عمل جميع الوظائف بشكل صحيح:

أولاً: متطلبات الخادم الخلفي (Backend)

المكوّنالإصدارالغرض
Python3.10+لغة البرمجة الأساسية للخادم
FastAPI + uvicorn0.110+ / 0.29+إطار الـ API والـ WebSocket
XGBoost2.0+نموذج التعلم الآلي للإشارات
scikit-learn1.3+معالجة البيانات (StandardScaler)
ccxt4.0+جلب بيانات Binance للكريبتو
yfinance0.2+جلب بيانات الأسهم والمعادن والعملات
pandas / numpyأحدث مستقرمعالجة البيانات الجدولية

ثانياً: متطلبات تطبيق Flutter (Frontend)

المكوّنالإصدارالغرض
Flutter SDK / Dart3.19+ / 3.3+إطار العمل الرئيسي للتطبيق
Android SDKAPI 21+ (Android 5.0)الحد الأدنى لنظام Android
iOS12.0+الحد الأدنى لنظام iOS
firebase_core / auth / firestoreأحدث مستقرالمصادقة وقاعدة البيانات السحابية
flutter_riverpod2.5+إدارة الحالة في التطبيق
go_router13.0+نظام التنقل بين الصفحات

ثالثاً: متطلبات الشبكة وFirebase

  • اتصال إنترنت مستقر (LTE/5G أو WiFi) لجلب الأسعار اللحظية والأخبار وتشغيل النماذج.
  • إمكانية الوصول إلى Binance API وYahoo Finance API (قد تتطلب VPN في بعض المناطق).
  • يتواصل التطبيق مع الخادم عبر HTTP على المنفذ 8000 مع آلية fallback بين عناوين IP متعددة.
  • مشروع Firebase نشط مع تفعيل Authentication وFirestore، وملفات التكوين المناسبة لكل منصة.

وصف الواجهة

شاشة تسجيل الدخول هي البوابة الرئيسية لنظام Market Insight، تُعرَض تلقائياً عند عدم وجود جلسة مستخدم نشطة.

العناصر الرئيسية:

  • شعار التطبيق: يُعرَض في الأعلى لتعزيز الهوية البصرية للنظام.
  • حقل البريد الإلكتروني: مُزوَّد بأيقونة بريد وتحقق من صحة التنسيق.
  • حقل كلمة المرور: يدعم إظهار/إخفاء النص عبر أيقونة العين.
  • رابط "نسيت كلمة المرور": يُوجّه لعملية الاسترداد عبر Firebase Auth.
  • زر Login: يُرسِل طلب المصادقة، وعند النجاح يُعيد التوجيه للرئيسية.
  • رابط إنشاء حساب: للمستخدمين الجدد غير المُسجَّلين.

الوظيفة التقنية:

تستخدم FirebaseAuth.signInWithEmailAndPassword(). عند الفشل تُعرَض رسالة خطأ واضحة للمستخدم.
