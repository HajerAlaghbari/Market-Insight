شكل 4.7 — هيكل جدول USERS في Firebase Firestore

توضيح حقول الجدول:

الحقلالنوعالقيدالوصف والغرض
uid PK string فريد · إلزامي المُعرّف الأساسي للمستخدم الذي تُنشئه Firebase Authentication تلقائياً عند تسجيل الحساب. يُستخدم كمفتاح أساسي لمسار الـ Firestore: users/{uid}
email string فريد · إلزامي عنوان البريد الإلكتروني للمستخدم المُسجَّل. يُستخدم للمصادقة عبر Firebase Auth ولعرضه في شاشة الإعدادات. تُتحقق Firebase من صحة تنسيقه وعدم تكراره تلقائياً.
createdAt timestamp إلزامي الطابع الزمني لتاريخ ووقت إنشاء الحساب (UTC). يُسجَّل تلقائياً لحظة التسجيل الأول ويُستخدم لتتبع نشاط المستخدمين وإحصاءات الاستخدام.

العلاقات مع الجداول الأخرى:

  • USERS ← يحفظ → FAVORITES: علاقة (1..∗) — مستخدم واحد يمكنه حفظ عدة رموز في المفضلة. تُخزَّن المفضلة كمجموعة فرعية: users/{uid}/favorites/{symbolCode}
  • USERS ← يولّد → SIGNAL_LOGS: علاقة (1..∗) — مستخدم واحد يمكنه توليد عدة إشارات تداول. تُخزَّن كمجموعة فرعية: users/{uid}/signal_logs/{id}

ملاحظة تقنية — Firebase NoSQL:

خلافاً لقواعد البيانات العلائقية (SQL)، لا تُخزَّن البيانات في Firestore ضمن جداول مترابطة ومُدمَجة، بل في مجموعات مستندات متداخلة (Nested Collections). هذا يعني أن بيانات كل مستخدم مُعزولة تلقائياً في مسار خاص به، ما يُسهّل تطبيق قواعد الأمان (Security Rules) ويُحسّن أداء الاستعلامات.

Firestore Collections Structure:
users/ (collection)
  {uid}/ (document)
    email: "user@example.com"
    createdAt: Timestamp
    favorites/ (subcollection)
    signal_logs/ (subcollection)

شكل 4.8 — هيكل جدول FAVORITES في Firebase Firestore

توضيح حقول الجدول:

الحقلالنوعالقيدالوصف والغرض
symbolCode PK string فريد ضمن المستخدم رمز الأصل المالي المُضاف للمفضلة (مثلاً: "BTCUSDT"، "XAUUSD"، "EURUSD"). يُستخدم كمُعرّف للمستند في Firestore. لا يمكن تكرار نفس الرمز للمستخدم ذاته.
uid FK string يُشير لـ USERS.uid مُعرّف المستخدم الذي أضاف هذا الرمز للمفضلة. مُضمَّن تلقائياً في مسار المستند: users/{uid}/favorites/{symbolCode}
name string اختياري الاسم الكامل للأصل المالي (مثلاً: "Bitcoin"، "Gold"، "Euro/US Dollar"). يُعرَض في شاشة قائمة الأسواق مع أيقونة النجمة ⭐ للرموز المُفضَّلة.

آلية إدارة المفضلة في التطبيق:

  • إضافة رمز للمفضلة: عند ضغط المستخدم على أيقونة ⭐ تُنفَّذ toggleFavoriteProvider التي تُنشئ مستنداً جديداً في users/{uid}/favorites/{symbolCode}.
  • إزالة رمز من المفضلة: ضغطة ثانية على ⭐ تحذف المستند المقابل من Firestore.
  • عرض المفضلة: يُراقب favoriteCodesProvider تدفق البيانات لحظياً ويُحدّث واجهة قائمة الأسواق فوراً عند أي تغيير.
  • المزامنة متعددة الأجهزة: بما أن البيانات في Firestore، فإن المفضلة تُزامَن تلقائياً عبر جميع أجهزة المستخدم.

قواعد الأمان (Security Rules):

match /users/{userId}/favorites/{symbolCode} {
  allow read, write: if request.auth.uid == userId;
}

تضمن هذه القاعدة أن كل مستخدم لا يمكنه قراءة أو تعديل مفضلة مستخدم آخر. يتحقق Firebase من صحة request.auth.uid تلقائياً.

الكيانات المرتبطة بهذا الجدول:

الجدول الأبنوع العلاقةالحقل المرتبطالوصف
USERSMany-to-One (∗..1)uid → USERS.uidكل سجل مفضلة ينتمي لمستخدم واحد فقط
